# PHP Blog

Простой блог на чистом **PHP 8.2** без фреймворков. Шаблонизатор **Smarty**,
база данных **MySQL**, стили на **SCSS**, окружение в **Docker**.

## Возможности

- **Главная** — категории, в которых есть статьи; для каждой выводятся 3 последних
  поста и кнопка «Все статьи».
- **Страница категории** — название, описание, список статей с сортировкой
  (по дате публикации / по количеству просмотров) и пагинацией.
- **Страница статьи** — полная информация о статье, счётчик просмотров и блок
  из 3 похожих статей (по общим категориям).
- Связь статья ↔ категория — **многие-ко-многим** (статья может быть
  в нескольких категориях).
- Сидер, наполняющий базу демо-данными и генерирующий обложки статей.

## Запуск

Нужен установленный Docker. Из корня проекта:

```bash
# 1. Собрать и поднять контейнеры (nginx + php-fpm + MySQL)
docker compose up -d --build

# 2. Установить зависимости PHP (Smarty)
docker compose exec php composer install

# 3. Заполнить базу демо-данными
docker compose exec php composer seed
```

Сайт будет доступен по адресу **http://localhost:8080**.

База данных доступна с хоста на порту **3307** (пользователь `blog`, пароль `blog`).

### Сидинг

Скрипт сидинга очищает таблицы и заново наполняет их данными из
`database/data.php`, а также создаёт SVG-обложку для каждой статьи:

```bash
docker compose exec php php database/seed.php
# или
docker compose exec php composer seed
```

## Маршруты

| URL                       | Страница            |
|---------------------------|---------------------|
| `/`                       | Главная             |
| `/category/{id}`          | Категория           |
| `/category/{id}?sort=views&page=2` | Сортировка и пагинация |
| `/article/{id}`           | Статья              |

## Стили (SCSS)

Скомпилированный CSS уже в репозитории (`public/assets/css/style.css`).
Чтобы пересобрать стили из исходников:

```bash
npm install
npm run build:css     # разовая сборка
npm run watch:css     # пересборка при изменениях
```

## Конфигурация

Настройки подключения к БД и приложения берутся из переменных окружения
(заданы в `docker-compose.yml`) со значениями по умолчанию в
`config/config.php`. Пример переменных — в `.env.example`.
